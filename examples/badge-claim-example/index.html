Badge Claim SDK — Examples

Every demo below embeds badge-claim.js with a different set of data-* attributes. View source on any page to see the exact one-line <script> tag a partner would paste.

Source modes

  • Pre-signed VC from data-src

    Partner hosts a pre-signed OBv3 credential; LearnCard wraps it in a VP at claim time. Cryptographic issuer of record: partner.

  • Unsigned VC from data-src (DIDAuth)

    Partner hosts an unsigned credential; LearnCard and the wallet perform a DIDAuth round-trip, then LearnCard signs with the holder's DID bound in as subject. Partner issuer is preserved as credentialSubject.awardedBy.

Display modes

  • data-mode="qr"

    QR code only — best for print or in-person display.

  • data-mode="button"

    Button only — best for mobile-first web flows.

  • Custom mount point via data-target

    Mount the widget into an existing element of your layout.

Use the host switcher at the top to flip every demo between Production, Staging, VetPass, and a local Netlify dev server at http://localhost:8888. Your selection is stored in localStorage and persists across pages.
