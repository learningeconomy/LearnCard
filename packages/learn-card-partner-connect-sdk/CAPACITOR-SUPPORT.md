# Capacitor & Localhost Support

## Overview

The Partner Connect SDK supports LearnCard running as a Capacitor mobile app on iOS and Android, as well as localhost development environments.

**Key Features:**
- Native app origins (Capacitor, Ionic, localhost) are recognized as valid **whitelist entries** for `lc_host_override` query parameter validation
- The `allowNativeAppOrigins` option (default: `true`) controls whether these origins are accepted in the whitelist
- **Multiple origins** can be configured by passing an array to `hostOrigin`, serving as a whitelist for `lc_host_override`

## How It Works

### Origin Resolution (at initialization)

The SDK determines a **single active origin** at initialization time using this hierarchy:

1. **`lc_host_override` query parameter** â€” if present and validated against the whitelist
2. **First configured `hostOrigin`** â€” fallback
3. **Default** â€” `https://learncard.app`

### Message Validation (at runtime)

Once the active origin is determined, **all incoming messages are validated with strict exact matching**:

```typescript
private isValidOrigin(eventOrigin: string): boolean {
  return eventOrigin === this.activeHostOrigin;
}
```

This means only messages from the single `activeHostOrigin` are accepted. There is no multi-origin acceptance at message time.

### Native App Origin Whitelist

Native app origins are relevant during **initialization only**, for validating the `lc_host_override` query parameter. When `allowNativeAppOrigins` is `true` (default), the following origins are accepted as valid `lc_host_override` values:

- `capacitor://localhost` (iOS Capacitor apps)
- `ionic://localhost` (legacy Ionic apps)
- `https://localhost` (with any port)
- `http://localhost` (with any port)
- `http://127.0.0.1` (with any port)

This is checked via `isOriginInWhitelist()` â†’ `isOriginNativeApp()`.

## Use Cases

### Production (Web)

```typescript
const learnCard = createPartnerConnect({
  hostOrigin: 'https://learncard.app'
});

// Active origin: https://learncard.app
// Accepts messages from: https://learncard.app ONLY
// Rejects: everything else (including capacitor://, localhost, etc.)
```

### Production (Capacitor Mobile App)

When LearnCard runs as a Capacitor app, the host sets `lc_host_override=capacitor://localhost` on the partner app's iframe URL:

```typescript
const learnCard = createPartnerConnect({
  hostOrigin: 'https://learncard.app',
  allowNativeAppOrigins: true  // default
});

// URL: https://partner-app.com/?lc_host_override=capacitor://localhost
// âś… capacitor://localhost is in whitelist (native app origin)
// Active origin: capacitor://localhost
// Accepts messages from: capacitor://localhost ONLY
```

### Staging with Override

```typescript
const learnCard = createPartnerConnect({
  hostOrigin: [
    'https://learncard.app',
    'https://staging.learncard.app'
  ]
});

// URL: https://partner-app.com/?lc_host_override=https://staging.learncard.app
// âś… staging.learncard.app is in configured whitelist
// Active origin: https://staging.learncard.app
// Accepts messages from: https://staging.learncard.app ONLY
```

### Local Development

```typescript
const learnCard = createPartnerConnect({
  hostOrigin: 'http://localhost:3000'
});

// Active origin: http://localhost:3000
// Accepts messages from: http://localhost:3000 ONLY
// Does NOT accept http://localhost:5173 or other ports
```

## Security Considerations

### âś… Strict Single-Origin Enforcement

At runtime, only the single `activeHostOrigin` is accepted for incoming messages. This provides strong security:
- No multi-origin acceptance at message time
- No wildcard or pattern matching
- Exact string equality check only

### âś… Capacitor Origins are Safe in the Whitelist

Capacitor origins (`capacitor://localhost`, `ionic://localhost`) are **unique to the Capacitor framework** and cannot be:
- Spoofed by malicious websites
- Set by arbitrary web pages
- Used by domains other than actual Capacitor apps

These origins are generated by the native mobile app container. Allowing them in the `lc_host_override` whitelist is safe because the LearnCard host controls which override value is set on the iframe URL.

### âś… Query Parameter Override is Whitelist-Gated

The `lc_host_override` query parameter is only accepted if its value is in the configured `hostOrigin` array (or is a native app origin when `allowNativeAppOrigins` is `true`). Invalid overrides are rejected with a console warning, and the SDK falls back to the first configured origin.

## Implementation Details

### Actual Code: `isValidOrigin` (Message Listener)

```typescript
private isValidOrigin(eventOrigin: string): boolean {
  // STRICT: Only accepts the single active origin
  return eventOrigin === this.activeHostOrigin;
}
```

### Actual Code: `isOriginInWhitelist` (Query Parameter Validation)

```typescript
private isOriginInWhitelist(origin: string): boolean {
  return (
    this.hostOrigins.includes(origin) ||
    (this.allowNativeAppOrigins && this.isOriginNativeApp(origin))
  );
}
```

### Actual Code: `isOriginNativeApp`

```typescript
private isOriginNativeApp(origin: string): boolean {
  const nativePatterns = [
    /^capacitor:\/\/localhost$/,
    /^ionic:\/\/localhost$/,
    /^https?:\/\/localhost(:\d+)?$/,
    /^https?:\/\/127\.0\.0\.1(:\d+)?$/,
  ];
  return nativePatterns.some(pattern => pattern.test(origin));
}
```
