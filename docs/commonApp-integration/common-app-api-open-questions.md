## Primary Open Question

The currently visible Swagger docs do not expose enough write endpoints to support the original prefill goal.

Most endpoints appear focused on:

-   authentication/token management
-   applicant lookup/details
-   FERPA link generation
-   counselor/recommender workflows
-   school/catalog metadata
-   application/recommendation status tracking

Open question for Common App:

> Does partner onboarding expose additional applicant write/prefill APIs that are not present in the public Swagger docs?

If not, LearnCard’s feasible integration may be limited to:

-   connecting applicant accounts
-   surfacing Common App status data
-   supporting FERPA/transcript/recommender workflows
-   powering college discovery from Data Catalog metadata

rather than directly pre-filling application sections from LearnCard credentials.
