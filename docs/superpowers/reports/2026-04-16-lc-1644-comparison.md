## Observations

### 1. Localhost numbers are tiny — production is different
The JIRA describes a 4–12s end-to-end user-perceived latency. Our localhost p50 total is ~100ms. The delta is production's external SA endpoint (~1–5s network hop). **Localhost benchmarks catch regressions but don't directly validate the "<4s end-to-end" target.**

### 2. Biggest local phases (for future optimizations)
- `logActivity` (17ms, p95=51ms) — writes activity log. Task 4 plans to parallelize with `sendBoost`.
- `saIssue` (15ms, dominated by `http` 13ms) — localhost SA. In production this is 1–5s and Task 3 (SA connection pooling) targets it.
- `sendBoost` (14ms, p95=25ms) — credential storage + notification queue.

### 3. What this benchmark CAN prove
- ✅ Backend has no regression after Task 2 (p50 within noise)
- ✅ Timing instrumentation (`PerfTracker`) is wired correctly and emits parseable JSON
- ✅ Docker log scraper extracts perf JSON successfully (22–24 samples captured per run)
- ✅ Harness works end-to-end and is automated

### 4. What this benchmark CAN'T prove
- ❌ Frontend claim-modal timing improvements from Task 2 (need `VITE_DEBUG_PERF=1` + browser-side `performance.mark` — the Playwright harness was built for this but has UI flakiness)
- ❌ Real SA latency wins from Task 3 (localhost SA responds in 13ms; production impact is ~100–300× this)
- ❌ End-to-end user-perceived flow (requires Partner-Connect iframe + claim UI)

## Known Issues with the Harness

1. **Sporadic UNAUTHORIZED failures** (~1–2 per 10 iterations): challenge reuse across the brain-client's batched request pattern. Worked around by rebuilding the tRPC client per iteration. Further investigation: could cache 95+ challenges at start and rotate.
2. **`p95=485ms` outlier** on after-task-2 run: one iteration spiked, likely brain-service cold-start behavior after `docker compose up -d --force-recreate --build brain`. Re-running on a warm stack should show smoother p95.
3. **Compose file path resolution:** pnpm filters set CWD to the service dir; resolving the compose file requires `git rev-parse --show-toplevel` (already handled). Brittle if used outside git context.

## Next Steps
